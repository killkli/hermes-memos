**Cron 排程**：每週一至五 07:00（台北時間）

## 系統架構

```
每週一至五 07:00 cron 觸發
    ↓
確認日期與市場狀態（是否交易日/週末/連假）
    ↓
並行蒐集 7 大產業新聞（分 3 批，共 7 個子代理）
    ↓
整合國際市場數據
    ↓
存檔至 ~/.nanobot/workspace/notes/morning_reports/{date}/產業晨報.md
    ↓
嘗試發送至用戶（Telegram/iMessage）
```

---

| 子代理 | 產業領域 | 搜尋關鍵字 |

| F | 綠能/電動車 | 電動車、氫能、太陽能、離岸風電 |
| G | 零售/消費 | 零售、內需、消費、服務業 |

## 日期判定邏輯

```
執行 date +%Y-%m-%d（Asia/Taipei）
    ↓
對照 taiwan_holidays_2026.md 判定
    ↓
├── 平日（週一至五）→ 標準晨報
├── 週末（週六日）→ 週末特刊（寬鬆日期限制）
└── 假期 → 停刊或特殊版
```

---

## 新聞驗證要求

**嚴格日期驗證**：每則新聞必須附上 `發布：YYYY-MM-DD`，且日期必須等於目標日期。

搜尋時加入日期限制：
```
"台積電 2026-04-21 site:cnyes.com OR site:udn.com"
```

## 錯誤處理

| 錯誤類型 | 發生情況 | 處理方式 |
|----------|----------|----------|
| HTTP 432 | Web search/excerpt 大面積故障 | 該產業標注 ⚠️ 資料有限 |
| HTTP 529 | 伺服器高負載 | 重試或以近期趨勢補充 |
| 子代理空資料 | 週末非工作日 | 寬鬆日期限制（±1天）並標注 |
| 存檔路徑錯誤 | `/root/` vs `/Users/johnchen/` | 使用 `~/.nanobot/` 避免路徑混淆 |

## 路徑工作區

```
~/.nanobot/workspace/notes/morning_reports/{date}/產業晨報.md
```

**⚠️ 注意**：正確路徑應為 `/Users/johnchen/.nanobot/`，而非 `/root/.nanobot/`。Agent 的 homedir 為 `johnchen` 非 root。

---

## 傳送機制

目前嘗試使用 `imessage` 或 `apple:imessage` 技能傳送，但：
1. 技能在 cron 環境可能不存在
2. 需要用戶電話號碼（無法自動取得）
3. 建議使用 cron 的自動 deliver 機制（`"deliver": "origin"`）而非手動傳送

---

## 國際市場數據

晨報包含美股收盤數據（道瓊、標普500、那斯達克、費半）。這些數據由主執行緒同步蒐集，不依賴子代理。
