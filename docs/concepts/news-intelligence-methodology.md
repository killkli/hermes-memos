## 一、核心理念

**目標**：將無結構的 Raw News（每日新間資訊流）轉化為有結構的 Teaching Material（教學素材）。

**挑戰**：新聞數量龐大、領域分散、深度不足。從新聞到知識，需要經過**挑選事實 → 識別知識點 → 設計教學**的轉化過程。

**自動化前提**：整個管線已高度自動化（每週一至五 07:00 cron 觸發 → 新聞蒐集 → 賢者分析 → 晨報生成），但教學轉化仍需要 LLM 人工干預。

## 二、新聞領域分類框架（9領域）

| 領域 | 定位 | 每週新聞量 | 教學潛力 |
|------|------|-----------|---------|
| **社會** | 公共議題、民生、人權 | 中（5-8則） | 高（爭議性議題多） |
| **政治** | 政府決策、選舉、兩岸 | 高（8-12則） | 中（易流於宣傳） |
| **科技** | AI、晶片、半導體 | 高（8-12則） | 高（台灣核心競爭力） |
| **財經** | 股市、產業、總經數據 | 中（5-8則） | 中（數據密集） |
| **教育** | 政策、校園、技職 | 低（3-5則） | 高（政策直接影響受眾） |
| **文化** | 藝術、文學、傳承 | 低（3-5則） | 高（故事性強） |
| **影視** | 戲劇、演唱會、偶像 | 中（5-8則） | 中（吸引力高但深度淺） |
| **音樂** | 產業、創作、節慶 | 低（3-5則） | 中（素材視覺化容易） |
| **國際** | 大國博弈、區域局勢 | 高（8-12則） | 高（影響台灣） |

## 三、Teachable Topic 挑選標準

並非所有新聞都適合做成教學影片。挑選標準：

### 3.1 知識密度標準

一個題材是否值得深入，取決於：

| 標準 | 問題 | 適合做教學？ |
|------|------|------------|
| **知識點數量** | 能否支撐 5-8 分鐘的教學內容？ | ≥3 個知識點 |
| **可延伸性** | 能否觸及其他相關主題？ | 高延伸性優先 |
| **受眾相關性** | 觀眾為什麼要關心這件事？ | 直接相關性優先 |
| **視覺化潛力** | 能否找到或生成相關圖片？ | 有圖片素材優先 |
| **避免重複** | 上週是否已做過類似主題？ | 不重複優先 |

### 3.2 知識點結構

每個教學影片的知識點結構：

```
核心概念（1個）
  └── 支撐事實（2-3個）
        └── 延伸問題（1個）
```

### 3.3 W16 實戰觀察：三領域空洞化

並非所有9個領域每週都有深度新聞。W16 觀察到：

| 領域 | 空洞化程度 | 典型情況 | 建議 |
|------|-----------|---------|------|
| **影視** | 高度空洞 | 全週都是娛樂緋聞/演唱會，無產業政策或創作深度 | 跨領域找議題，或降低該領域集數比重 |
| **音樂** | 高度空洞 | 全週都是演唱會消息，無產業趨勢或創作報導 | 同上 |
| **教育** | 中度空洞 | 所有項目均與科技/文化重疊，無獨立政策 | 以科技/文化領域的新聞為主，教育為輔 |

### 3.4 新聞來源品質分級

| 來源 | 等級 | 適用場景 |
|------|------|---------|
| 中央社（CNA） | ⭐⭐⭐ 最佳 | 正式教學影片首選，時間乾淨無舊聞 |
| 關鍵評論網、蕃新聞 | ⭐⭐ 良好 | 補充觀點，須注意立場 |
| Aggregator（Yahoo、ETtoday） | ⭐ 可用但需核查 | 常有舊聞重複推送，時間戳混亂，須人工核查 |

## 四、Script 撰寫方法論

### 4.1 對話腳本格式

每個教學場景由**兩人對話**構成：

```json
{
  "scene": 1,
  "image_prompt": "英文描述，用於 MiniMax 圖生成",
  "narration_A": "角色A的台詞",
  "narration_B": "角色B的台詞"
}
```

### 4.2 圖片 Prompt 原則

- **語言**：英文（送給 MiniMax image-01 模型）
- **內容**：具體場景描述，避免抽象概念
- **風格**：教學影片風格，16:9 橫向
- **人物**：避免直接描繪真實人物

### 4.3 敘述設計原則

- **角色分工**：A 負責提問和引導，B 負責解釋和補充
- **避免機器感**：不用「讓我們來看」「你知道嗎」等罐頭开场白
- **1秒法則**：圖片出現後 1 秒再開始語音，避免語音與圖片同時出現
- **字幕同步**：最終 MP4 搭配 .srt 字幕檔（FFmpeg subtitles filter 不支援 MP4 mux，需 sidecar 檔）

## 五、新聞事實核查框架

適用於有爭議的新聞敘述或數據聲明：

| 核查層次 | 工具 | 用途 |
|----------|------|------|
| 網路交叉查詢 | web_search | 同一事件是否有多個獨立來源 |
| 官方數據核對 | web_extract | 政府統計、營收數字 |
| 專有名詞確認 | 工具書/RAG | 人名、機構、術語 |
